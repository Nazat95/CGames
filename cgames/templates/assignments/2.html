Lists

In python, we use lists to store multiple elements. In general, all elements in the lists must be of the same type. For syntax, we use brackets, [ and ] for lists. For example, take a look at this list declaration {% call utils.code('python') %} myList = ["python", "java","c sharp", "ruby", "perl"] {% endcall %} This list, which I've decided to call myList, is a list of five strings. Notice that I don't have to write (and should not write) list myList = ['python', 'java','c sharp', 'ruby', 'perl']
As you can see, my list has 5 elements. I can call len on myList to get the size of the list I've created. In this case len(myList) == 5.

Now let's say I want to add 'kuda' to myList. To do this, I use the append() function, like this: myList.append('kuda') and the result is: ['python', 'java','c sharp', 'ruby', 'perl', 'kuda']
But, let's say I want 'kuda' at the beginning of my list. To do this, append() won't help me. Instead, I'll have to use the insert() method. insert() takes two arguments. the index where you want the new element to go, and then the element you want to add, like this!

myList.insert(0,'kuda') This tells my Python program to add kuda to myList at index 0. The result is: ['kuda','python', 'java','c sharp', 'ruby', 'perl']

'kuda' is at index 0, and all other elements have been moved to the right(each element's index has been increased by 1)
Notice that I don't have to tell Python to make myList bigger to make room for the new element 'kuda' that I'm adding. It knows to do this and takes care of it automatically. If I want to remove an element I use the function remove(). remove() takes one argument, the element that you want to take out of the list. So, if I want to remove 'java' I'd write myList.remove('java') then if I call print(myList) this would be printed to the console: ['kuda', 'C++', 'python', 'ruby', 'perl'] Another handy function is the sort() function. If I call myList.sort(), this will sort the elements of myList into abc order. Also, myList.reverse() is used to flip a list, like this! {% call utils.code('python') %} myList = ['python', 'java','c sharp', 'ruby', 'perl'] >>>myList.sort() ['c sharp', 'java','perl', 'python', 'ruby'] >>>print(myList) ['c sharp', 'java', 'perl', 'python', 'ruby'] >>>myList.reverse() ['ruby', 'python', 'perl', 'java', 'c sharp'] {% endcall %}

If Statements

In Python, there are times when we only what lines of code to execute some of the time -- not all the time. To control when lines of code run, we can use if statements. Let's look at some code to see how these if statements work!

{% call utils.code('python') %} myList = ['bills','dolphins','jets','patriots'] if (len(myList) == 4): print('This list has four elements') else: print('hello, world') Output >>> This list has four elements {% endcall %} So, why does the first statement get printed, but not the second statement? Well, the first if statement evaluates to true, since myList does have a length of 4. Then, the code skips the else branch, since that branch only gets read if the if statement evaluates to false.


Let's look at another example {% call utils.code('python') %} myList = ['bills','dolphins','jets','patriots'] if myList[3] == 'bills': print('bills are last') elif myList[3] == 'dolphins': print('dolphins are last') elif myList[3] == 'jets': print('jets are last') else: print('patriots are last') Output >>> patriots are last {% endcall %} Why? This time, all of the if statements evaulate to false. So, none of the code in the if statements are executed. Instead, the else statement is executed.

For Loops

Sometimes we want to iterate over, or read each element in a list. To accomplish that, we use for loops, like this! {% call utils.code('python') %} myList = ['python', 'java','c sharp', 'ruby', 'perl'] for x in range(0,len(myList)): print(myList[x]) >>> python >>> java >>> c sharp >>> ruby >>> perl {% endcall %} Here, the elements of myList get printed out, one at a time. This is because when we enter the loop we start with x = 0, so we print(myList[0]). Then we re-enter the loop. This time x=1 so we print myList[1]
This process keeps going until x = 5. x is no longer in the range we specified!

NOTE: in Python, the x in range loop is NOT inclusive. So, for x in range (0,2) will loop twice, when x is assigned to 0 and again when x is assigned to 1. x=2 would not meet our loop conditions, so we would exit the loop. Along the same lines, len(myList) == 5 so when x=5 we exit the loop {% endcall %} {# TODO: Modularize #} {% call section('quiz', 'Quiz') %}
